HTML5 인라인 SVG를 지원 합니다.

What is SVG?

  • SVG는 확장 가능한 벡터 그래픽(Scalable Vector Graphics)을 의미합니다
  • SVG는 웹을 위한 벡터 기반 그래픽( vector-based graphics)을 정의하는 데 사용됩니다
  • SVG는 XML 형식의 그래픽을 정의
  • SVG 그래픽은 어떻게 확대되거나 크기가 조정 된 경우에도 품질을 잃지 않는다
  • SVG 파일에 있는 모든 요소 및 모든 속성은 애니메이션 될 수 있습니다
  • SVG는 W3C 권장 사항입니다

SVG Advantages

(JPEG 및 GIF와 같은) 다른 이미지 형식에 대한 SVG를 사용의 장점은 다음과 같습니다:

  • SVG 이미지는 텍스트 편집기로 생성하고 편집 할 수 있습니다
  • SVG 이미지는 검색, 색인, 스크립트화 및 압축 할 수 있습니다
  • SVG 이미지는 확장가능하다.
  • SVG 이미지 해상도에 관계없이 고품질로 인쇄 될 수 있다
  • SVG 이미지는 질의 저하없이 확대 할 수 있습니다

직접 HTML 페이지에 삽입한는 SVG

HTML5 에서 직접 HTML 페이지에 SVG 요소를 포함시킬 수 있습니다:

Example

<!DOCTYPE html>
<html>
<body>

<svg width="300" height="200">
  <polygon points="100,10 40,180 190,60 10,60 160,180"
  style="fill:lime;stroke:purple;stroke-width:5;fill-rule:evenodd;" />
</svg>

</body>
</html>

SVG 와 캔버스의 차이점

SVG는 XML에서 2D 그래픽을 기술하는 언어입니다.

캔버스(Canvas)는 자바스크립트 로 2D 그래픽을 즉시 그립니다.

SVG는 XML에 기반을 두고 있으며, 이는 모든 요소가 SVG DOM 내에서 사용할 수 있음을 의미합니다. 이러한 요소에 자바 스크립트 이벤트 핸들러를 첨부 할 수 있습니다.

SVG 에서 각각 그려진 모양은 객체로 기억되며, 객체의 속성이 변경되는 경우, 브라우저가 자동으로 모양을 다시 렌더링 할 수있다.

Canvas는 픽셀 단위로 렌더링됩니다. 캔버스에서는 그래픽이 그려지고 나면, 브라우저에 의해 잊혀진다. 위치를​​ 변경해야 하는 경우 그래픽이 적용되었을 수있는 모든 개체를 포함하여 전체 장면은 다시 그려야 할 필요가있다.

캔버스와 SVG의 비교

아래의 표는 캔버스와 SVG 간의 차이점을 보여줍니다:

Canvas SVG
  • 해상도에 의존
  • event handlers 를 지원하지 않음
  • 않좋은 text rendering 능력
  • 결과 이미지를 .png 또는 .jpg 로 저장가능
  • graphic-intensive games 에 매우 적절
  • 해상도와 무관
  • event handlers 지원
  • large rendering areas (Google Maps) 응용에 매우 적절
  • 복잡할 경우 rendering이 느림 (anything that uses the DOM a lot will be slow)
  • game applications 에 적당하지 않음
