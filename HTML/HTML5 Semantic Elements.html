시맨틱 = 의미

시맨틱 요소 = 의미있는(의미론적) 요소.

시멘틱 요소란?

시맨틱 요소는 브라우저와 개발자 모두에게 그 의미를 명확하게 설명합니다.

비-시멘틱(non-semantic) 요소 : <div> 와 <span> - 그 내용에 대해 아무것도 알려주지 않습니다.

시맨틱(semantic) 요소: <form>, <table>, 과 <img> - 그 내용을 분명히 정의해 줍니다.

HTML5의 새로운 시맨틱 요소들

오늘날, 기존 웹 사이트의 대부분은 다음과 같은 HTML 코드가 포함되어 있습니다: 탐색 링크, 머리글 과 바닥글을 표시하기 위해서 <div id="nav">, <div class="header">, 또는 <div id="footer"> 를 사용합니다.

HTML5 웹 페이지의 여러 부분들을 깨끗이 정의하기 위해 새로운 시맨틱 요소를 제공합니다:
  • <header>
  • <nav>
  • <section>
  • <article>
  • <aside>
  • <figure>
  • <figcaption>
  • <footer>
  • <details>
  • <summary>
  • <mark>
  • <time>

HTML5 <section> Element

<section> 요소는 문서의 section을 정의한다.

W3C의 HTML5 문서에 따르면: " 섹션(section)은 일반적으로 제목이 있는 주제를 갖는 내용들의 그룹으로 정의합니다."

Example

<section>
  <h1>WWF</h1>
  <p>The World Wide Fund for Nature (WWF) is....</p>
</section>

Try it yourself »

HTML5 <article> Element

<article> 요소는 독립적인 자체적으로 만족되는 내용을 지정합니다.

글/기사(article)는 그자체로 의미를 가질 수 있어야 하며, 웹 사이트의 나머지 부분과는 독립적으로 배포 할 수 있어야한다.

<article> 요소가 이용 될 수 있는 예:

  • 포럼 게시물
  • 블로그 포스트
  • 뉴스 기사
  • 논평

Example

<article>
  <h1>Internet Explorer 9</h1>
  <p>Windows Internet Explorer 9 (abbreviated as IE9) was released to
  the  public on March 14, 2011 at 21:00 PDT.....</p>
</article>

Try it yourself »

HTML5 <nav> Element

<nav> 요소는 탐색 링크(navigation links)의 집합을 정의.

<nav> 요소는 탐색 링크의 큰 블록을 위한 것이나, 문서의 모든 링크 <nav> 요소 내부에 있어야 하는 것은 아니다.

Example

<nav>
<a href="/html/">HTML</a> |
<a href="/css/">CSS</a> |
<a href="/js/">JavaScript</a> |
<a href="/jquery/">jQuery</a>
</nav>

Try it yourself »

HTML5 <aside> Element

<aside> 요소는 주요 내용을 제외한 내용의 정의하며 사이드 바 등에 배치된다.

aside 컨텐츠는 주변 컨텐츠와 관련되어 있어야 한다.

Example

<p>My family and I visited The Epcot center this summer.</p>

<aside>
  <h4>Epcot Center</h4>
  <p>The Epcot Center is a theme park in Disney World, Florida.</p>
</aside>

Try it yourself »

HTML5 <header> Element

<header> 요소는 문서 또는 섹션 헤더를 지정합니다.

<header> 요소는 소개 내용에 대한 컨테이너로 사용되어야 한다. .

당신은 하나의 문서에 여러 <HEADER> 요소를 포함 할 수 있습니다.

다음은 기사의 헤더를 정의한다:

Example

<article>
  <header>
    <h1>Internet Explorer 9</h1>
    <p><time pubdate datetime="2011-03-15"></time></p>
  </header>
  <p>Windows Internet Explorer 9 (abbreviated as IE9) was released to
  the  public on March 14, 2011 at 21:00 PDT.....</p>
</article>

Try it yourself »

HTML5 <footer> Element

<footer> 요소는 문서나 구역의 바닥 글을 지정합니다.

<footer> 요소는 포함하는 요소에 대한 정보를 포함해야합니다.

바닥글(footer)은 일반적으로 등 문서의 작성자, 저작권 정보, 이용 약관에 대한 링크, 연락처 정보를 포함한다.

하나의 문서에 여러 <footer> 요소를 포함 할 수 있습니다.

Example

<footer>
  <p>Posted by: Hege Refsnes</p>
  <p><time pubdate datetime="2012-03-01"></time></p>
</footer>

Try it yourself »

HTML5 <figure> and <figcaption> Elements

<figure> 태그 같은 그림, 도표, 사진, 코드 샘플 등과 같이 자족적인(self-contained) 내용을 지정합니다.

<figure> 요소의 내용이 주된 흐름과 관련되어 있지만, 그 위치는 독립적이며, 제거된 경우에도 그 문서의 흐름에 영향을 주지 않아야 한다.

<figcaption> 태그는 <figure> 요소에 대한 캡션을 정의합니다.

<figcaption> 요소는 <figure> 요소의 첫 번째 또는 마지막 내용으로 배치 할 수 있습니다.

Example

<figure>
  <img src="img_pulpit.jpg" alt="The Pulpit Rock" width="304" height="228">
  <figcaption>Fig1. - The Pulpit Pock, Norway.</figcaption>
</figure>

Try it yourself »

이러한 시맨틱 요소를 현재 사용할 수 있습니까?

위의 설명된 (<figcaption> 제외) 모든 요소가 블록 요소이다.

이러한 요소들이 이전 버전의 브라우저에서 제대로 작동하도록 하기 위해, 스타일 시트에 display 속성을 다음과 같이 설정합니다:

header, section, footer, aside, nav, main, article, figure
{
display: block;
}

Semantic Elements in HTML5

Below is an alphabetical list of the new semantic elements in HTML5. The links goes to our complete HTML5 Reference.

Tag Description
<article> Defines an article
<aside> Defines content aside from the page content
<details> Defines additional details that the user can view or hide
<figcaption> Defines a caption for a <figure> element
<figure> Specifies self-contained content, like illustrations, diagrams, photos, code listings, etc.
<footer> Defines a footer for a document or section
<header> Specifies a header for a document or section
<main> Specifies the main content of a document
<mark> Defines marked/highlighted text
<nav> Defines navigation links
<section> Defines a section in a document
<summary> Defines a visible heading for a <details> element
<time> Defines a date/time
